# travel-maps 手机与桌面行为

travel-maps 以 `760px` 为手机/桌面布局分界。CSS 与 JavaScript 必须共用
`(max-width: 760px)`，修改断点时需要同步检查 `css/style.css`、`js/app.js` 与
`js/map.js`。

## 行为对照

| 项目 | 手机（≤ 760px） | 桌面（> 760px） |
| --- | --- | --- |
| 城市贴纸 | 全部 190 个城市始终显示；目的地贴纸视觉尺寸为桌面的 50%，透明点击命中区保持原尺寸 | 全部 190 个城市按标准尺寸显示 |
| Home 标记 | 始终显示 | 始终显示 |
| 时间图例 | 默认收起，可用左下角按钮展开/再次收起 | 默认展开，也支持收起 |
| 缩放控件 | 隐藏放大、缩小按钮，保留恢复 100% | 显示放大、缩小、恢复 100% |
| 角落安全区 | 使用 `100dvh` 与 `env(safe-area-inset-*)`，避免 Safari 工具栏、圆角和 Home Indicator 遮挡 | 同样使用安全区，普通桌面环境 inset 为 0 |

用户手动展开或收起图例后，当前页面生命周期内不再由断点变化覆盖该选择。

## 实现位置

- `index.html`：`viewport-fit=cover`、图例折叠按钮与内容容器。
- `css/style.css`：动态视口、弹性页面结构、安全区、图例折叠动画及手机缩放按钮差异。
- `js/app.js`：图例默认状态、ARIA 状态与用户切换逻辑。
- `js/map.js`：手机目的地贴纸的 50% 视觉缩放，Home 与点击命中区不缩小。

## 回归检查

1. iPhone Safari 工具栏展开时，左下图例和右下恢复按钮都完整位于可视区域内。
2. 390 × 844 手机尺寸首次打开时，图例收起，只显示一个缩放恢复按钮。
3. 手机始终显示 190 个城市，目的地贴纸尺寸约为桌面的 50%，点击区域不缩小。
4. 桌面首次打开时图例展开，190 个城市和三个缩放按钮全部显示。
5. 图例按钮的 `aria-expanded`、文案与实际展开状态一致。
6. `prefers-reduced-motion: reduce` 下图例直接切换，不播放展开/收起动画。
